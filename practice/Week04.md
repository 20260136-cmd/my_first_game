# 4주차 실습 정리
## 오늘 한 것

- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용
AABB는 회전된 객체에 대해 가장 큰 축에 정렬된 직사각형을 사용하여 불필요하게 넓은 영역을 감지하는 반면,
OBB는 회전된 객체의 정확한 모양에 맞춰 최소 직사각형으로 충돌을 감지합니다.
원형 충돌은 중심과 반지름을 기준으로 충돌을 감지해, 모서리 충돌을 정확하게 처리합니다.

### 원형 vs AABB
AABB는 수평/수직 축에 정렬된 충돌 감지 방식으로, 모서리 충돌에서 정확도가 떨어집니다.
원형 충돌은 중심과 반지름을 기준으로 모서리 충돌을 정확하게 감지합니다.


### AABB vs OBB
AABB는 회전된 객체에 대해 가장 큰 축에 정렬된 직사각형을 사용, 불필요하게 넓은 영역을 감지합니다.
OBB는 회전된 객체의 정확한 모양에 맞춰 최소 직사각형으로 충돌을 감지합니다.

## AI와의 대화에서 배운 것
캐릭터 스프라이트가 찌그러졌을 때,비율 계산을 통해 이미지의 원본 비율을 유지하고,
smoothscale()을 사용하여 품질을 높였습니다.
이미지 크기 조정 시, 원본 크기에 맞게 Rect의 크기를 동적으로 조정하여,
왜곡이나 찌그러짐 없이 이미지가 정확하게 표시될 수 있도록 했습니다.


## 내 게임에 적용한다면
정밀한 조종이 필요한 게임이라면 OBB방식을 선택해 더 정확한 충돌감지를 하고,
만약 게임 오브젝트가 많다면 계산이 복잡한 OBB방식 대신 AABB를 선택할 것이다.
그리고 게임에 원형 오브젝트가 있다면 원형 충돌을 선택할 것이다.
