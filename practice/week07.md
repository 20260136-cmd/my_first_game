# 7주차 실습 기록

##적용한 기법
screenshake,무적 프레임

## 사용한 AI 프롬프트 (요약)
기능 구현: "전체 화면 기능을 넣고, 창 크기에 맞춰 게임 화면이 비율을 유지하며 확대되게 해줘."
버그 수정: "차 소리가 안 나는 문제와 게임오버 UI가 오른쪽으로 쏠린 현상을 고쳐줘."
밸런스 조정: "난이도가 너무 높으니 레벨 구성을 한 단계씩 밀고, 1레벨은 더 쉽게 새로 만들어줘."

## AI 답변에서 도움이 된 것
화면 흔들림(Screenshake) 구현: 피격 시 shake_amount 값을 부여하고 이를 최종 출력 좌표(pos_x + ox, pos_y + oy)에 더해 박진감 넘치는 타격감을 연출함.
사운드 트리거 최적화: passed_sound_played 플래그를 활용해 자동차가 플레이어 곁을 지날 때 소리가 중복 실행되지 않고 정확히 한 번만 출력되도록 개선함.

## AI 답변을 수정하거나 버린 것
단순 중앙 배치: 초기에는 창 크기만 커지고 게임 화면은 작게 유지되었다.
이를 해결하기 위해 smoothscale을 이용해 화면 크기에 맞춰 게임 화면을 가득 키우면서도, 
원본 비율을 잃지 않도록 보정하는 방식으로 코드를 전면 교체함.

UI 고정 좌표: 800x700 기준으로 설정된 UI 좌표가 큰 화면에서 잘리는 문제를 발견하여, 
텍스트가 항상 화면 안쪽에 안전하게 들어오도록 여백 값을 직접 수정함.

## 적용 결과
전체 화면 전환 시 비율 유지가 완벽하며, 화면 흔들림 효과 덕분에 플레이어가 
데미지를 입었다는 사실을 시각적으로 강렬하게 인지할 수 있음.
